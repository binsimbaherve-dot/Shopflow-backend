# Security Policy



## Supported Versions



ShopFlow is currently in active pre-launch development. Security fixes are applied to the main branch only.



| Branch | Supported |

| ------ | --------- |

| main   | Yes |

| dev    | Development only - not for production use |



## Reporting a Vulnerability



If you discover a security vulnerability in ShopFlow, please report it responsibly:



1. Do not open a public GitHub issue for security vulnerabilities.

2. Email binsimbaherve@gmail.com with a description of the vulnerability, steps to reproduce, and potential impact.

3. You should receive an acknowledgment within 72 hours.

4. We will investigate and aim to provide an initial assessment within 7 days.

5. Once resolved, we will coordinate disclosure timing with you if you wish to be credited.



## Scope



This policy covers the ShopFlow backend (Shopflow-backend) and any associated frontend or infrastructure repositories under the binsimbaherve-dot GitHub account related to the ShopFlow project.



## Our Commitment



- We will not take legal action against researchers who report vulnerabilities in good faith and in accordance with this policy.

- We will keep you informed of progress toward resolution.

- We follow a threat model documented in THREAT_MODEL.md (STRIDE methodology) as part of our ongoing security process.
